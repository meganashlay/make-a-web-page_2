Lesson 3: Adding CSS Style

Cascading Style Sheets

  • CSS or Cascading Style Sheets is code written to control the "style" of HTML elements
  • CSS controls HTML by selecting an element and telling it what rules apply to it. It does this to every like element.
  • To apply CSS to only certain elements, use class selectors (.class-name)
  • Rules can apply to all like elements or only certain ones by using the appropriate selector.
  • There are 3 ways to include CSS to your page:
    1. Write CSS in the head of your HTML.
    2. Link your HTML to a seperate CSS file.
    3. Write your style inline with your HTML.

IDs and Classes

  • IDs should only be used once. They work similarly to classes by applying style rules. They help keep down repetition. They are helpful as bookmarks
  • Classes can be used more than once. Use them to style mutliple elements the same way instead of writing the same code multiple times. They save time, make code easier to read, and avoid repetition.
  • It is important to avoid repetition because it will save you time, make your code easier to read, and it will help the page load faster.

Code, Test, Refine

Use this routine when coding pages:
  1. Look for natural boxes.
  2. Look for repeated styles and semantic elements.
  3. Write your HTML.
  4. Apply styles (from BIGGEST to smallest).
  5. Fix things

Errors in HTML and CSS

It is very easy to make mistakes when coding. It is important to remember to close all open tags. For example, leaving a div tag open can completely mess up the look of your page. A good process to follow when debugging is:
  • Check for syntax errors. (Did you leave any tags open? Or forget a semicolon?)
  • Use a validation tool. (They can help find errors that you might have missed.)
  • Prevent cross-browser issue. (Often times browsers are not completely compatible with each other. It is best to anticipate this and include code for all browsers.)
  • Explore you page with Dev Tools. (Dev tools can help you break down you page visually and help you make adjustments.)

Avoiding Repetition

It is important to avoid repetition in your code. Avoiding repetition will keep file size down and save time while writing your code. Use classes to avoid repetition. Why write the same CSS for each div when you could write one class for all of them?

Lesson 2: Variables and Strings

What Is A Variable

A variable is a name that refers to a value. In Python, we can use any sequence of letters, numbers and underscores to make a variable name (just as long as it doesn't start with a number). Here are some examples of valid variable names:
  • megan
  • book_name
  • _character_name
  • child_3

Variables are useful because they make your code easier to read and write. Wouldn't you rather rather write "number_of_pages" than "87834598794", especially if you have to use that particular number several times? This will also decrease the risk of accidentally typing the number wrong and throwing everything off.

They are also useful because we can change the value of them. For instance we can make the value increase like this:
days=5
days=days+1
After this code finishes days will actually be 6.

Assignment Statements

To introduce a new variable, you use an assignment statement. You set the variable equal to a value. For instance: megan=7. Then for the rest of your code (or until you give "megan" a new value) megan will actually be 7. If you say: print megan, it will print out 7. The = sign does not mean the same thing in Python that it does in math. In math you are looking at an equation and the answer will always be the same. In programming you are setting the value of the variable. It can be anything you want it to be. In this case, think of the = sign more like an arrow. You are giving the value on the right to the variable on the left.

Printing with Strings

There is a difference between:
  • print 2+2 and
  • print "2"+"2"
The first one is an equation and will print the result of 2 plus 2 which is 4.
The second, however, is two strings being concantenated together. It will print 22.
